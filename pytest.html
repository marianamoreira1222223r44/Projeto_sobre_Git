Como utilizar o PyTest?

Pytest é uma estrutura de testes Python que se originou do projeto PyPy. Ele pode ser usado para escrever vários tipos de testes de software, incluindo testes unitários, testes de integração, testes de ponta a ponta e testes funcionais. Suas características incluem testes parametrizados, luminárias e reescrito de afirmação.



As luminárias Pytest fornecem os contextos para testes, passando em nomes de parâmetros em casos de teste; sua parametrização elimina o código duplicado para testar vários conjuntos de entrada e saída; e suas declarações de afirmação reescritas fornecem saída detalhada para causas de falhas.



Pytest foi desenvolvido como parte de um esforço de pacotes de terceiros para resolver as deficiências do módulo embutido do Python. Originou-se como parte do PyPy, uma implementação alternativa do Python ao CPython padrão. Desde sua criação, no início de 2003, a PyPy tem tido uma grande ênfase nos testes. PyPy tinha testes unitários para código recém-escrito, testes de regressão para bugs e testes de integração usando o conjunto de testes do CPython. Em meados de 2004, uma estrutura de testes chamada utest surgiu e os contribuintes do PyPy começaram a converter os casos de teste existentes para utest. Enquanto isso, no EuroPython 2004, uma biblioteca padrão complementar para testes, chamada STD, foi inventada. Este pacote expôs os princípios, como a reescrita de afirmação, do que mais tarde se tornaria pytest. No final de 2004, o projeto std foi renomeado para py, std.utest tornou-se py.test, e a biblioteca py foi separada de PyPy. Em novembro de 2010, pytest 2.0.0 foi lançado como um pacote separado do py. Ele ainda era chamado de py.test até agosto de 2016, mas após o lançamento do pytest 3.0.0 o ponto de entrada recomendado da linha de comando tornou-se pytest.
